## PPTX DEMO

**STEP 1 — Download the Skill Live (1.5 min)**

- Open Claude's official skills repo (or show the install method — `npx skills add pptx` or manual copy)
- Drop the skill folder into `.cursor/skills/` right in front of the audience
- Briefly expand the folder tree in Cursor's sidebar: "SKILL.md, scripts folder, assets folder — that's the three layers from the presentation"

**STEP 2 — Quick Anatomy Peek (1 min)**

- Open `SKILL.md` for 15 seconds — point at the YAML frontmatter and say "this description is what tells the agent when to load it"
- Open one asset file (e.g., a template or style reference) — "this is the institutional memory layer"
- Don't over-explain — the presentation already covered this

**STEP 3 — Trigger It (3 min)**

- Type a natural prompt: *"Create a 5-slide presentation about the benefits of remote work"* (or any simple, universally relatable topic)
- Let the agent work. Narrate lightly: "It loaded the SKILL.md... now it's using the template from assets... running the build script..."
- This is where progressive disclosure becomes visible — callback to Slide 7

**STEP 4 — Show the Output (1.5 min)**

- Open the generated `.pptx` — flip through the slides on screen
- Let the visuals do the talking: title slide, content slides with structure, consistent styling
- One line: "One sentence, five polished slides, no formatting by hand"

**STEP 5 — Wrap & Transition to Q&A (1 min)**

- "What you just saw: we downloaded a skill, dropped it in a folder, typed one prompt, and got production output. That's the entire workflow."
- Open it up for questions

1. **Pre-test the pptx skill in Cursor** end-to-end at least twice before demo day

4. **Keep Cursor's sidebar visible** so the audience can see the folder structure throughout
